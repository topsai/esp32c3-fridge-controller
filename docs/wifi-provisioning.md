# Wi-Fi 配网

冰箱控制器可以通过手机为 ESP32-C3 保存家庭 Wi-Fi。配网只影响网络功能；温度采样、压缩机控制、传感器故障保护和风扇停机后
5 分钟散热在配网期间仍运行。

## 何时出现配网

配网热点只会在以下情况出现：

1. 开机时 ESP32-C3 没有已保存的 Wi-Fi；
2. 同时按住升温键和降温键至少 5 秒，主动清除旧 Wi-Fi。

已有 Wi-Fi 但路由器关闭、密码被路由器修改或信号中断时，控制器只在后台重连，等待多久都不会自动开放配网热点。

## 首次配网

1. 给控制器上电。
2. OLED 显示 `WiFi Setup`、`Fridge-Setup-XXXXXX`、以 `P:` 开头的一次性热点密码和 `192.168.4.1`。
3. 在手机 Wi-Fi 设置中连接 OLED 显示的热点。
4. 输入 OLED 上 `P:` 后面的临时热点密码。
5. 手机通常会自动弹出配置页；没有弹出时，浏览器访问 `http://192.168.4.1/`。
6. 选择家庭 Wi-Fi，输入家庭 Wi-Fi 密码并保存。
7. 连接成功后配置热点关闭，OLED 恢复正常温度界面。

临时热点密码每次配网重新随机生成，只显示在 OLED，不写入 HIL、日志或 Git。

## 60 秒窗口

每次配网窗口最多 60 秒。60 秒内没有成功保存时：

- 配网热点自动关闭；
- OLED 点亮并恢复正常温度界面；
- 本次开机不再自动开启配网；
- 冰箱继续按正常逻辑运行；
- 如果仍没有保存 Wi-Fi，下次重新上电又会获得一个新的 60 秒窗口和新临时密码。

错过窗口时也可以不重启，直接同时按住两个温度按钮 5 秒重新打开一个 60 秒窗口。

## 更换路由器或 Wi-Fi

1. 同时按住升温键和降温键 5 秒。
2. 按住期间两个按键不会改变设定温度。
3. 控制器只清除 Wi-Fi 凭据，不清除温度设定值。
4. 根据 OLED 上的新热点名称和临时密码重新配网。

若正在执行 OTA，双键配网请求会被忽略；必须等待 OTA 完成并重新启动后再操作。

## 错误密码和网络故障

- 在配网页面输入错误家庭 Wi-Fi 密码：页面在 60 秒窗口内保持可用，可重新输入；保存时最多等待约 5 秒验证连接。
- 已保存的家庭 Wi-Fi 后来无法连接：状态在 `connecting` 与 `retry_wait` 之间重试，不开启热点。
- 路由器恢复：控制器重新连接，OTA 服务随后恢复为 `ready`。
- 必须修改错误的已保存密码：双键长按 5 秒，重新配网。

## 安全边界

- 配网热点使用每次随机生成的 WPA2 密码，并在 60 秒后关闭。
- 配网页面菜单只提供 Wi-Fi 选择和退出；固件更新、擦除、重启及管理路径被拒绝。
- 家庭 Wi-Fi 凭据由 ESP32 Wi-Fi/NVS 保存，不进入项目文件。
- OTA 仍使用独立的 OTA 密码，详见 [`ota.md`](ota.md)。
- 配网是局域网现场维护入口，不应把设备配置热点转发到公网。

## HIL 状态

HIL/Python 仪表板显示：

- `wifi_state`：`no_credentials`、`unconfigured_idle`、`connecting`、`connected`、`retry_wait` 或 `portal`；
- `wifi_provisioning`：当前是否正在配网；
- `wifi_ap_ssid`：仅配网时显示热点名称；
- `wifi_connected`、`ota_ip`：家庭 Wi-Fi 的连接状态和地址。

仪表板不会显示家庭 Wi-Fi 名称、家庭 Wi-Fi 密码、热点临时密码或 OTA 密码。

## 实机验证提示

自动测试验证状态转换、手势边界、代码集成和固件容量，但不能代替真实手机、路由器、OLED 和实体按键验证。首次测试应断开市电负载，
确认 HIL 安全锁、继电器有效电平和风扇行为后再连接压缩机。
